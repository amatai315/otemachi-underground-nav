# 大手町地下移動ナビ PoC 要件定義

## 目的
東京都ハッカソン用プロトタイプ。大手町駅周辺の地下街を模した簡易3D空間で、出発地(改札/出口)と目的地を選ぶと経路をハイライト表示する。

## スコープ(個人PoC / スマホ実機で動けばOK)
- 対象デバイス: **スマートフォン(縦画面)前提**。PC向けの汎用Webレイアウトは不要
- 対象エリア: 大手町駅構内(周辺の主要な改札・出口を含む範囲)
- リアルタイム測位は実装しない(GPS/PDR/画像照合は不要)
- **地図データは実データを使用する**(架空のモックデータは使わない)
- **GitHub Pagesで公開する**(静的ホスティングのみ。サーバーサイド処理は持たない)

## データソース(実データ)
- **東京駅周辺屋内地図オープンデータ(令和2年度更新版)**
  - 提供: 国土交通省「高精度測位社会プロジェクト」、公開先: G空間情報センター
  - URL: https://www.geospatial.jp/ckan/dataset/mlit-indoor-tokyo-r2
  - 形式: Shapefile。通路・部屋の範囲(基盤地図)、歩行空間ネットワークデータ(ノード/エッジ)、トイレ・エレベータ等のPOIを含む
  - 対象範囲: 東京駅を中心に東西約1km・南北約2km(大手町駅はこの範囲に含まれる)
  - ダウンロードにはG空間情報センターのユーザー登録が必要(無償)
- Shapefileの座標系・属性構造を確認し、必要な範囲(大手町駅周辺)を抽出して使用する(GeoPandas/ogr2ogr等でGeoJSON変換すると扱いやすい)

## 技術スタック(提案)
- フロントエンドのみ、Vite + Vanilla JS(TypeScript不要)
- 3D描画: Three.js(スマホでのタッチ操作に対応したOrbitControls相当。ピンチズーム・ドラッグ回転)
- データ前処理: Shapefile→GeoJSON変換(Python/GDAL等)は**ビルド前のローカル作業**として実施し、変換済みGeoJSONを静的ファイルとしてリポジトリに含める(GitHub Pagesはサーバー処理を持てないため、実行時に変換や外部API呼び出しは行わない)
- 階層(フロア)ごとに分けて押し出しポリゴン化
- 経路探索: ダイクストラ法(自前実装で十分。データ内の歩行空間ネットワークをそのままグラフとして利用)
- デプロイ: Vite の `base` を GitHub Pages 用リポジトリパスに合わせて設定し、`vite build` の成果物を `gh-pages` ブランチ or `docs/` フォルダとして公開

## データ設計
- 上記オープンデータの歩行空間ネットワーク(ノード/エッジ)をそのまま経路探索用グラフとして利用
- ノードの属性から改札名・出口名等のPOI名称を取得し、UIの選択肢に使う(実在の名称なので説得力が出る)
- 通路・部屋ポリゴンはThree.jsで押し出し表示する際の元データとして使用

## 機能要件
1. 起動時に3D空間(押し出しポリゴンで通路・壁・床を表現)を表示
2. 「出発地」「目的地」を選ぶUI(タップで選べるリスト/ボタン一覧、ノード一覧から選択)
3. 選択後、ダイクストラで最短経路を計算し、3D空間上で経路をハイライト(色変更 or ライン表示)
4. カメラはタッチ操作(ドラッグで回転、ピンチでズーム)で自由に視点変更可能

## UI
- **スマホ縦画面(375〜430px幅程度)を基準にレイアウト**。PC画面での見え方は考慮しない
- 上部 or 下部に固定のコントロールパネル(出発地/目的地選択、経路探索ボタン)、残りの領域に3Dビュー
- タップ操作前提(ボタン・リストは指で押しやすいサイズに)
- スタイリングは最低限、動作優先

## Out of Scope(今回やらない)
- 実測位(GPS/PDR/画像照合)
- PC/タブレット向けのレスポンシブ対応
- バリアフリールート等の重み付け
- サーバーサイド処理・API(GitHub Pagesは静的ホスティングのみのため)

## 完了条件
- GitHub Pagesで公開したURLにスマホでアクセス → 出発地/目的地を選択 → 3D空間上に経路がハイライトされる、一連の動作が確認できること
